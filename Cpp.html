  1. Data Types:

    - Data type is a term that is used to show the kind of data value or the type of data that is expected to be handled.

    C/C++ supports three types of data types:

    1. Built-in Data types.
    2. Derived Data types.
    3. User-defined Data types.
    1. Built-in Data type:

      These are also known as fundamental data types. These are defined inside the compiler and are not composed of any other data type.

      C/C++ Support the Following Fundamental Data types:

      • int
      • float
      • char
      • double
      • long int
      • boolean
      • void
      • string//Note: only C++ supports string data type.
    2. Derived Data Types:

      These are the Data types that are composed of fundamental data types.

      Eg: Array, Pointers and Reference.

    3. User-defined Data Types:

      These are the data types that are defined by the user.

      Eg: Structures, Union, Class in C++.

  2. Variables:

    - Variable is a namespace to hold a value to perform operations.

    - Variables can be initialized in three ways: 

    int a;              //Uninitialized variable.
int a = 10;         //Initialized variable.
int a = 10*7;       //Dynamically Initialized variable.

  3. Header Files:

    - Header Files are special pre-defined file in C/C++, for some special purpose. The '#' Symbol in the Header File declaration is called preprocessor directive. It indicates that the specified header files in the program are included before compilation.

    - Syntax: 

    //Some Commonly used Header Files:
'#'include<iostream>          //for C++.
'#'include<stdio.h>           // for C.

  4. Input/Output:

    - Input and Output can be managed by using printf() and scanf() in both the languages, however C++ also allows the use of cin>> and cout<< for this purpose. Traditionally, scanf() and printf() are slightly faster than cin>> and cout<<. However, it does not makes much of a difference.

    - Syntax for C and C++: 

    '#'include<stdio.h>
void main()
{
    printf("Hello World!");
    int a;
    printf("Enter a Number: ");
    scanf("%d",&a);
    printf("Entered Number is: %d\n",a);
}

    - Syntax for C++: 
    '#'include<iostream>
using namespace std;
int main()
{
    cout<<"Hello World!";
    int a;
    cout<<"Enter a Number: ";
    cin>>a;
    cout<<"Entered Number is:"<<a;
    return 0;
}

  5. Operators:

    - Operators are sign that trigger some computation or action when applied to variables and other objects in C/C++.

    Operators available in C/C++ are:

    • Arithmetic Operator:

      - They are used to perform Arithmetic calculations like addition, subtraction, division, modulus.

      Eg: + , - , / , * , %

    • Relational Operators:

      - They are used to check the relation between two operands.

      Eg: > , < , >= , <= , == , !=

    • Logical Operators:

      - They are used for logic.

      Eg: && , || , !

    • Increment and Decrement: 
      int a=5;
cout<<a++;          //prints 5. Since its post increment.
cout<<a;            //prints 6.
cout<<++a;          //prints 7. Since its pre increment.
cout<<a--;          //prints 7. Since its post decrement.
cout<<--a;          //prints 5. Since its pre decrement.
    • Conditional Operator:

      - It is an alternative of if condition and is used to perform an action based on some condition.

      - Syntax:

      #include<stdio.h>
void main()
{
    int a=1;
    a==1? printf("Yes"): printf("No.");         //Syntax. Its prints Yes as output.
}
  6. IF-ELSE Statements:

    - The if statement is the most extensively used conditional statement.
    - Syntax: 

    if(condition)
{
    ...         //statements to execute if statement is true.
}
    Note:

    - A logical statement always evaluate to true/false
    - Any value in C/C++ other than zero (positive/negative) is considered to be true, whereas a zero(0) is considered to be false.
    - If there exists only one line of program statement under if() scope then we may omit curly braces{} . - If statement can stand alone, a else statement is not compulsory. But for every else statement a if statement is compulsory.

    - IF-ELSE Syntax:
    if(condition)
{
    ...         //Code to Execute if the condition is true.
}
else
{
    ...         //Code to Execute if the condition is false.
}
    - ELSE-IF Syntax:
    if(condition)
{
    ...         //code if true.
}
else if(condition)
{
...             //code if second condition is true.
}
else
{
    ...         //code if non of the conditions were true.
}
  7. Switch Statement:

    - The Switch Statement provides an alternative for the if-else-if construct. It is a multi-way branching statement.

    - Syntax:

    switch(expression)
{
    case val_1: 
        ...             //Statement block 1 if expression == val_1.
        break;
    case val_2:
        ...             //Statement block 1 if expression == val_2.
        break;
    case val_n:
        ...             //Statement block 1 if expression == val_n.
        break;
    default:
        ...             //Statement block if non of the condition is matched.
        break;
}
    Note:

    - Every case needs to have a break statement because when it is encountered, the control jumps to the line of code following the switch statement, i.e. outside the body of switch statement. If a case statement does not include a break statement, then the control continues right on the next case statement until either a break is encountered or end of switch is reached. This situation is called fall through.

  8. For Loop:

    - The For loop is used when we have prior knowledge of how many times the particular group of statements is to be executed.

    - This loop is called an entry control loop because condition is checked before entering into the scope of the loop.

    //Syntax.
for(expression 1; expression 2; expression 3)
{
    statement blocks;
}

/*
expression 1 is the initialization part.
expression 2 is the test expression or condition.
expression 3 is update or increment/decrement expression.
*/
  9. While Loop:

    - A while Loop tests for its ending condition before executing the statements enclosed in the loop body even the first time.

    - It is also an entry controlled loop.

    //Syntax.
while(condition)                //condition or terminating condition.
{
    statement block;
}
    Note:

    - The while loop can accept expression and not just condition, the following are all legal expression:

    • while(x= x-1)
    • while(n--)
    • while(count -= i)
  10. Do-While Loop:

    - The do-while loop allows its body to be executed one or more times. A do-while loop executes atleast once.

    - The body of the do statement continues to execute as long as the result of the condition in the while statement remains true.

    - The do-while loop is also called exit controlled loop.

    //Syntax:
do{
    statement block;
    ...
}while(condition);
  11. Break and Continue Statements:
    Break Statement:

    - The break statement ends execution of the nearest enclosing loop or conditional statement in which it appears.

    - Control passes to the statement following the ended statement, if any.

    //Syntax:
break;
    Continue Statement:

    - The continue statement cause the program to skip the rest of the loop in the current iteration, as if the end of the statement block has been reached, causing it to jump to the start of the following iteration.

    - It does not execute the statements, which are after continue in the block of statements and transfer the control to the new iteration in the loop.

    //Syntax:
continue;
  12. goto and exit() Statements:
    goto Statement:

    - The goto statement unconditionally transfer control to the statement labeled with the specified identifier.

    - The labeled statement designated by identifier must be in the current function.

    //Syntax:
goto identifier_name;
...
identifier_name:
    ...     //statement to execute;
    exit() Statement:

    - exit() is an inbuilt function and is used to terminate the execution of the program at any point at any stage.

    - In C this function is included inside <process.h> header file.

    //Syntax:
exit(value);

/*
Note: 
    - Here, value is an integer variable or value, depending on the following:
        -- Using exit(1); returns a value of 1 to the system indicating that an error must have occured. This process is often used for error trapping.
        -- Using exit(0); exits a program without any error message, but other codes indicate that the system can handle the error messages.
*/
    Eg program illustrating goto and exit() statements:
    #include<iostream>
using namespace std;
int main()
{
    cout<<"Program Begins.\n";
    int a=1;
    if(a==1)
        goto yo;
    cout<<"Hello world.\n";
    yo:
        cout<<"Goto statement worked!\n";
    if(a==1)
        exit(0);
    cout<<"exit didn't worked.";
    return 0;
}

/*
Output:
Program Begins.
Goto statement worked!
*/
  13. Arrays:

    - An Array is a collection of the same type of variables that are referred by a common name, which are stored in memory sequentially.

    - Therefore, an integer array holds integers, a character array holds characters.

    There are two types of Arrays:
    • One- Dimensional Array.
    • Two- Dimensional Array.
    One-Dimensional Array:

    - An Array is a continuous memory location holding similar type of data in a single row or single column is called one-dimensional array.

    /*
Syntax:
    data_type array_name[ array_size ];
*/

//eg:
    int arr[10];

//Valid Syntaxes for initializing one-dimensional array are:
    int arr[5] = {1,2,3,4,5};
    int arr[] = {1,2,3,4,5};
    int arr[5];
    Two-Dimensional Array:

    - A two-dimensional array is a continuous memory location holding similar type of data arranged in row and column format like matrix structure.

    /*
Syntax:
    data_type  array_name[no_of_rows][no_of_columns];
*/

//eg:
    int arr[5][5];

//legal Syntax for 2D Arrays:
    int arr[3][3] = {{1,2,3},{4,5,6},{7,8,9}};
    int arr[3][3] = {1,2,3,4,5,6,7,8,9};
    int arr[][3] = {1,2,3,4,5,6,7,8,9};
//Note: Number of Columns is compulsory to be declared.
  14. Functions:

    - A Function is a named unit of a group of executable program statements, this unit can be invoked from other part of the program as an when required.

    - There are mainly two types of Functions:

    • Library Functions.
    • User Defined Functions.
    Library Functions:

    - Library Functions are pre-defined functions in programming system.

    - These are also known as in-built functions.

    - Eg: printf(), scanf(), cin, cout, etc.

    User Defined Functions:

    - These are functions that are defined by the programmer as per the requirement.

    - There are three main parts of a User-Defined Function, they are:

    1. Function Prototype:

      - It is the declaration of the function that tells the program about the type of value returned by the function, number and type of each argument taken by the function.

      - Function Prototype is exactly same as the function definition header except its body.

      - Note: It is not compulsory unless and until the function definition is above the main() function.

      //Syntax:
    return_type function_name( arrgument_data_type arrgument_name);
    2. Function Definition:

      - It describes the action to be performed by the function.

      - Main difference between function definition and function Prototype is that a function Prototype has a semicolon in the end of its header while a function definition does not.

      //Syntax:
return_type func_nm( d_t var_nm_1, d_t var_nm_2 )
{
    ...         //Body of the Function.
}
    3. Function Call:

      - It is used to access function that is defined by the programmer.

      //Syntax:
    func_nm(parmeters);
  15. Formal and Actual Parameter:
    Formal Parameter:

    - The Parameters defined inside a function definition are called Formal Parameters. These are the names used inside a function, which are also referred as its arguments.

    //eg:
int sum(int x, int y)
{
    return (x+y);
}

/*
    Here, x and y are Formal parameters and they tell the compiler to accept two integer values at the time of function call.
*/
    Actual Parameter:

    - The Parameters that appear in the function call statement are called Actual Parameters. They may be an constant values, variables or a complex expression.

    - When a function uses call by reference method, then the Formal Parameter becomes a reference to the Actual Parameter. In Such case any changes that are made to the Formal Parameter are reflected in the Actual Parameters.

    //eg: the function call statement of above sum function looks like;
int main()
{
    int a=5,b=6;
    cout<<sum(a,b)<<endl;        //Here a and b are Actual Parameters.
    cout<<sum(8,9)<<endl;        //Here 8 and 9 are the Actual Parameters.
    cout<<sum(8-4,9/3)<<endl;        //Here 8-4 and 9/3 are the Actual Parameters.
    return 0;
}
  16. Call by Value and Call by Reference Methods:
    Call By Value Method:

    - The Call by Value Method copies the value of the actual parameter into the formal parameter, i.e, the function creates its own copy of argument values and then uses them.

    - If any changes are done on formal parameters, it will not affect the actual parameters.

    //Illustration code:
#include<iostream>
using namespace std;
int square(int x)            //Here, x is formal parameter.
{
    x*=x;
    return x;
}
int main()
{
    int a=5;
    int sq= square(a);      //Call by Value Method. where a is actual parameter.
}
/*
    Note:
        -> Here the value of a remains unchanged.
        -> Also, the value of a is copied to x.
*/
    Call By Reference Method:

    - In this method, no separate copy is created for the formal parameter, i.e, formal parameters shares the same location of actual parameters.

    - Hence, any change made to the formal parameter is automatically reflected back to the actual parameter.

    //Illustration Code:
#include<iostream>
using namespace std;
void square(int *x)          //Here x is Formal parameter and will point to the location of actual parameter.
{
    (*x) *= (*x);
}
int main()
{
    int a=5;
    square(&a);             //Actual parameter's location is passed to the function.
    cout<<a<<endl;          //This line will print 25 since a's value has got changed.
}
  17. Inline Functions:

    - In Inline functions, the compiler replaces the function call statement with the function code itself. This process is called "Expansion" and then compiles the entire code.

    - Thus, the compiler does not have to jump to another location to execute the function.

    - Inline functions are designed to speed up the programs.

    - Coding of normal function and Inline function is similar except that in Inline function the function definition starts with the keyword inline.

    Note:

    - The Difference between normal and inline function is that both have different compilation process.

    - An inline function definition should be placed above all the functions that call them.

    - The functions should be inlined only when they are small.

    - There is no point inlining a large function as there would be heavy memory penalty.

    - Memory Penalty: If 10 times an inline function is called then there will be 10 copies of the function inlined into the code.


    Inline Function does not work in the following conditions:

    - For functions that return a value and are having a loop or a switch statement or a goto statement.

    - For functions not returning value, but its return value exists.

    - If function contains static variable.

    - If the function is recursive.

    //Illustration Program:
#include<iostream>
using namespace std;
inline void max(int a,int b)            //Notice the keyword inline in the header of the definition.
{
    cout<<(a>b?a:b);
}
int main()
{
    int x,y;
    cin>>x>>y;
    max(x,y);                           //prints larger number out of x and y.
}
/*
    Note:
        -> Here, max() is an inline function so it would not be called in main() during execution, rather its code would be inserted into main() and then compiled.
*/
  18. Recursive Function:

    - A function that invoke itself in its own body is known as a Recursive Function.

    - A recursive function should always contain a terminating condition.

    //Eg:
int print(int n)
{
    if(n==1)
    {
        cout<<1<<endl;
        return 0;
    }
    else
    {
        cout<<n<<endl;
        return print(n-1);
    }
}
/*
    -> The Above function will print N numbers in decreasing order starting from n to 1.
*/
  19. Structures:

    - A Structure is a Heterogenous Data type. It is a collection of variables of different data types under one name.

    //General Structure Format:

//Syntax 1:
struct structure_name{
    <data-type> member1;
    <data-type> member2;
    - - - - - - - - - - -
    - - - - - - - - - - -
    <data-type>member_n;
};

//Example of Syntax 1 :
struct student{
    int rollno;
    char nm[10];
};

//For this structure we can declare structure variable as follows:
    struct student s1,s2,s3,s4,s5;

//Syntax 2.

struct structure_name
{
    <data-type> member1;
    <data-type> member2;
    - - - - - - - - - - -
    - - - - - - - - - - -
    <data-type>member_n;
}struct_var_list;

/*
    -> Above syntax shows, structure definition with structure variable declaration.
    -> Struct_var_list is list of structure variables.
*/

//Example of Syntax 2 :

    struct student
    {
        int rollno;
        char nm[10];
    }s1,s2,s3,s4,s5;

    - A Macro refers, to a #define defination that is used to define words to enhance readibility and understnadability. Before compilation the preprocessor replaces every occurance of macro as per the defination.

    //Eg:
    #define ll long long int

//The above line will replace all ll by long long int within the program
  21. Const:

    - A const is used over a variable to make its value a constant and thus its value cannot be changed or modified during the program run.

    Difference between macro and a const:

    - Both macro and const define constants, however #define can only define simple constants, while const can define almost any type of C/C++ constant, including things like structure classes.

    - A macro is preprocessor directive and is text-replace during compilation whereas a const is type identifier having constant value.

    - A const object is subjected to the scope rules for variable whereas a #define is not.

    - The compiler does not type-check a macro whereas a const is definitely type-checked.

    //Syntax Eg:
    const int a=50;

// Now the value of variable 'a' is a constant and cannot be modified anywhere in the program. Its value will remain 50 throughout.
